## 1. 引言

### 1.1 目的

本文档旨在完整描述“校园互助服务平台（CampusHub）”的功能与非功能需求。平台目标是为南京大学（及类似高校）学生提供一个统一、可信、高效的互助服务渠道，解决当前需求分散在微信群、QQ群等渠道导致的匹配效率低、信任缺失等问题。

### 1.2 范围

系统将支持以下核心业务模块：用户管理、需求发布与浏览、订单流程管理、评价与信用体系、消息通知、管理后台。智能匹配与推荐、即时通讯作为可选或进阶功能。

### 1.3 定义与缩略语

| 术语 | 解释 |
|------|------|
| 求助方 | 发布互助需求的用户 |
| 帮助方 | 接单并完成任务的用户 |
| 信用分 | 基于历史交易评价计算的用户信誉值 |
| 积分 | 平台内虚拟奖励单位，可兑换实物或服务 |

### 1.4 参考文献

- 《软件工程与计算Ⅱ》大作业总览（2026）
- 校园互助平台调查问卷原始数据（N=51）
- 四份深度访谈记录（2026.4.13-4.15）

## 2. 总体描述

### 2.1 用户角色

| 角色 | 说明 |
|------|------|
| 普通用户 | 已通过学号认证的在校学生，可发布需求、接单、评价 |
| 管理员 | 平台运营人员，负责用户管理、内容审核、数据统计 |

注：同一用户可同时作为求助方和帮助方。

### 2.2 用户需求摘要

基于问卷与访谈，用户最关注的功能为：

- **跑腿类**（快递代取、外卖、占座） – 高频提及
- **失物招领** – 高频提及
- **二手交易** – 高频提及
- **咨询/答疑**（学习、选课、留学经验） – 高频提及
- **找搭子**（运动、组队、拼车） – 中频提及
- **信息公告牌**（跨届引导、课程经验） – 访谈中强调

用户对以下方面存在明确担忧或偏好：

- **信任**：学号认证被多数接受（约70%），但约30%担心隐私泄露。
- **纠纷处理**：用户期望有明确的责任判断标准，支持照片、聊天记录、位置签到等多重证据。
- **酬劳**：约80%愿意接受平台积分，但要求积分具有实际兑换价值（如奶茶、月卡、现金提现）。
- **收费**：多数倾向完全免费或自愿捐款，少数接受小额手续费或广告。
- **线上交流**：绝大多数认为需要接单前沟通，平台应提供私聊或“打招呼”机制。

### 2.3 约束条件

- 开发周期：10周，团队3人（原4人角色合并）
- 技术栈：不限，建议前后端分离
- 必须完成 MVP 功能（见3.1节）
- 所有代码、文档需提交至 GitHub 仓库，遵循 GitFlow 分支策略

## 3. 具体需求

### 3.1 功能需求（按模块）

以下功能来源于问卷及访谈中用户明确提及的需求，并按优先级（P0必做、P1应做、P2可选）分类。

#### 模块一：用户管理（P0）

| ID | 需求描述 | 来源 |
|----|----------|------|
| U-01 | 用户使用**学号+密码**注册，需验证为在校学生（统一认证或学号+姓名匹配） | 问卷Q4、访谈R1 |
| U-02 | 用户可选择补充个人资料：昵称、头像、学院、联系方式（手机/微信） | 大作业文档 |
| U-03 | 支持密码修改和找回功能（通过绑定邮箱或手机） | 常规需求 |
| U-04 | 用户可查看自己的信用分、历史订单、评价记录 | 问卷Q13、访谈R3 |
| U-05 | 用户可设置是否允许他人通过学号/姓名搜索到自己的信息 | 访谈R2（隐私保护） |

#### 模块二：需求发布与浏览（P0）

| ID | 需求描述 | 来源 |
|----|----------|------|
| D-01 | 发布互助需求：标题、详细描述、分类（快递代取、学习辅导、二手交易、活动组队、失物招领、咨询、其他）、地点、期望完成时间、报酬类型（积分/现金/无偿） | 问卷Q2、Q10 |
| D-02 | 需求支持多标签，便于检索（如#急单# #需排队#） | 问卷建议 |
| D-03 | 需求列表支持按分类、时间、距离（位置）、报酬方式筛选排序 | 问卷Q2、访谈R4 |
| D-04 | 需求详情页展示发布者信用分、历史评价（匿名）、地理位置（模糊化） | 问卷Q13 |
| D-05 | 用户可编辑、下架自己发布的需求（未接单前） | 常规 |
| D-06 | 需求超过24小时无人接单可自动提醒发布者或推荐相似帮助方 | 访谈R2 |

#### 模块三：订单流程管理（P0）

| ID | 需求描述 | 来源 |
|----|----------|------|
| O-01 | 帮助方可对未接单的需求点击“接单”，状态变为“进行中” | 大作业文档 |
| O-02 | 任务完成后，帮助方提交“完成”并上传凭证（照片、文字说明） | 问卷Q13、访谈R1 |
| O-03 | 求助方确认完成，系统自动将报酬（积分/现金托管）转给帮助方 | 问卷Q13 |
| O-04 | 若求助方48小时内未确认，系统自动确认完成（需双方事先同意） | 问卷建议 |
| O-05 | 订单状态机：已发布 → 已接单 → 进行中 → 待确认 → 已完成 / 争议中 | 大作业文档 |
| O-06 | 任何一方可在订单完成前发起“争议”，平台介入仲裁 | 问卷Q13、访谈R1 |
| O-07 | 订单历史记录永久保留，用户可查看自己参与的所有订单 | 大作业文档 |

#### 模块四：评价与信用体系（P1）

| ID | 需求描述 | 来源 |
|----|----------|------|
| E-01 | 订单完成后，双方可匿名互评（1-5星 + 文字评价） | 问卷Q13 |
| E-02 | 信用分 = 基础分 + 完成订单数×系数 - 被投诉次数×扣分 | 常规 |
| E-03 | 信用分划分为不同等级（如青铜、白银、黄金），在个人主页显示 | 大作业文档 |
| E-04 | 用户可选择不逐单评价，平台每季度或每半年收集一次评价（避免评价疲劳） | 访谈R3 |
| E-05 | 违规行为（如爽约、欺诈）经管理员核实后扣除信用分并记录 | 问卷Q5 |

#### 模块五：消息通知（P1）

| ID | 需求描述 | 来源 |
|----|----------|------|
| N-01 | 用户收到站内消息：接单通知、状态变更通知、评价通知 | 大作业文档 |
| N-02 | 支持“打招呼”机制：帮助方可向求助方发送一条预定义消息（回复前仅限一条） | 访谈R2 |
| N-03 | 双方接单后可进行一对一私聊（文字+图片） | 问卷Q12 |
| N-04 | 用户可屏蔽陌生人的私聊请求 | 访谈R2 |

#### 模块六：智能匹配与推荐（P2，可选）

| ID | 需求描述 | 来源 |
|----|----------|------|
| R-01 | 基于需求分类、用户历史行为（常接快递类）推荐匹配 | 大作业文档 |
| R-02 | 按距离排序推荐附近的需求 | 访谈R4 |

#### 模块七：管理后台（P2，可选）

| ID | 需求描述 | 来源 |
|----|----------|------|
| A-01 | 管理员可查看、禁用违规用户 | 大作业文档 |
| A-02 | 管理员可审核被举报的需求、评论 | 问卷Q5 |
| A-03 | 数据统计看板：日活、订单量、分类分布、信用分分布 | 大作业文档 |
| A-04 | 管理员处理争议时，可查看聊天记录、位置签到、上传凭证 | 问卷Q13 |

#### 模块八：积分与报酬（P1）

| ID | 需求描述 | 来源 |
|----|----------|------|
| P-01 | 用户可使用积分发布“悬赏”需求，积分可充值或通过帮助他人获得 | 问卷Q11、访谈R1 |
| P-02 | 积分可兑换：小黄车月卡、奶茶折扣、小红包、AI会员等 | 访谈R1 |
| P-03 | 平台不强制现金交易，但允许用户自行协商现金报酬（需在订单中注明） | 问卷Q11 |
| P-04 | 积分不可直接提现，但可通过兑换商品间接变现（避免金融风险） | 访谈R1（用户倾向提现，但平台设计需谨慎） |

### 3.2 非功能需求

| ID | 类别 | 需求描述 | 来源 |
|----|------|----------|------|
| NF-01 | 性能 | 核心页面（列表、详情）加载时间 < 2 秒 | 大作业文档 |
| NF-02 | 安全 | 密码加密存储（bcrypt），防SQL注入、XSS | 大作业文档 |
| NF-03 | 可用性 | 核心操作路径（发布→接单→完成）不超过3步 | 大作业文档 |
| NF-04 | 可维护性 | 代码结构清晰，关键模块有单元测试 | 大作业文档 |
| NF-05 | 隐私 | 用户学号、姓名等敏感信息仅在订单完成后对交易对方可见，默认脱敏 | 问卷Q4、访谈R3 |
| NF-06 | 合规 | 平台内容需符合校园规定，禁止代考、代课等违规需求 | 问卷Q5 |

### 3.3 外部接口需求

- 地图定位接口（如高德/腾讯地图）用于选择位置和距离排序。
- 短信/邮件接口用于密码找回（可选）。
- 统一认证接口（若校方提供）用于学号验证。

## 4. 附录：需求来源统计摘要

### 4.1 问卷关键数据（N=51）

| 问题 | 主要结论 |
|------|----------|
| 最重要的功能 | 资料分享、校园资讯、失物招领、跑腿、咨询 |
| 学号认证态度 | 70%支持（认为安全、唯一），30%担忧隐私 |
| 收费方式偏好 | 完全免费（35%）、自愿捐款（30%）、部分功能收费（25%）、手续费（10%） |
| 使用意愿（1-10分） | 平均约7.2分，中位8分 |
| 积分接受度 | 约80%愿意接受积分，20%只要现金 |
| 线上交流必要性 | 95%认为需要 |

### 4.2 访谈核心发现

- **R1**：积分可兑换实物（奶茶、月卡、会员）能有效激励；纠纷处理倾向“严格判定失败”；广告不接受。
- **R2**：需要“可编辑精选公告牌”积累经验；希望设置陌生人打招呼限制；愿意展示个人专长关键词。
- **R3**：评价疲劳——希望按季度收集反馈；关注匿名性和历史记录时效性。
- **R4**：定位距离推荐；自定义地图选点；报酬参考价由平台提供。
