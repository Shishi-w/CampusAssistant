# 校园互助服务平台需求规格说明书（SRS）
**依据标准：IEEE 830-1998**  
**适用对象：南京大学校园互助服务平台**

---

## 1. 引言

### 1.1 编写目的
本需求规格说明书旨在明确南京大学校园互助服务平台的功能需求、性能需求、用户特征、约束条件及系统接口，为开发、测试、管理和维护人员提供统一依据。

### 1.2 项目背景
随着高校学生对便捷生活服务、学习互助、资源共享的需求不断提升，校园内需要一个安全、可信、可追溯的互助平台，支持学生之间的任务发布、接单、交易与协作。本平台旨在提升校园资源利用效率，促进互助文化建设。

### 1.3 定义与缩写
- **用户**：南京大学在校学生
- **任务**：用户发布的互助需求，如代取快递、学习辅导等
- **接单者**：接受任务并完成的学生
- **平台**：校园互助服务平台（Web + 移动端）
- **NJU**：Nanjing University

### 1.4 参考资料
- IEEE 830-1998《软件需求规格说明书推荐实践》
- 《南京大学学生管理规定》
- 高校校园服务平台案例分析

## 2. 总体描述

### 2.1 产品前景
平台定位为校园内部互助生态系统，提供任务发布、接单、二手交易、学习辅导、组队匹配等功能。未来可扩展至校园活动组织、志愿服务、AI 学习助手等模块。

### 2.2 产品功能概述
- 任务发布与接单
- 二手交易市场
- 学习辅导与答疑
- 组队匹配（课程/竞赛/运动）
- 用户信用体系
- 消息通知与聊天
- 校园认证与安全机制

### 2.3 用户特征
- **本科生**：需求多样，使用频率高
- **研究生**：偏向学习辅导、学术互助
- **留学生**：需要语言友好界面
- **校园管理者**：负责审核、监管、数据统计

### 2.4 约束条件
- 必须使用校园统一身份认证（SSO）
- 数据需存储在校内服务器或合规云服务
- 遵守《个人信息保护法》
- 交易需符合校园管理规定
- 平台需支持高并发

### 2.5 假设与依赖
- 学生均可使用校园网或移动网络
- 校方提供认证接口
- 校园地点信息可获取

## 3. 具体需求

---

## 3.1 功能需求

### 3.1.1 用户注册与认证
- 支持校园统一身份认证（SSO）
- 首次登录需绑定手机号
- 用户可设置头像、昵称、专业、年级等信息

### 3.1.2 任务发布模块
支持任务类型：
- 快递代取
- 跑腿服务
- 学习辅导
- 二手求购
- 组队需求

任务字段：
- 标题、类型、描述
- 价格/报酬
- 截止时间
- 地点
- 附件（可选）

系统需支持：
- 敏感词审核
- 任务状态管理（待接单、进行中、已完成、已取消）

### 3.1.3 接单与任务流程
- 用户可浏览任务并接单
- 接单后双方可聊天
- 完成任务后接单者提交完成证明
- 发布者确认完成并评价
- 系统更新双方信用分

### 3.1.4 二手交易模块
- 发布商品（图片、描述、价格）
- 买家留言/私聊
- 交易状态管理（在售、已预定、已售出）
- 举报违规商品

### 3.1.5 学习辅导模块
- 发布求助（课程答疑、作业辅导、考研咨询）
- 辅导者可设置擅长科目
- 支持图文/语音交流

### 3.1.6 组队匹配模块
- 课程小组
- 科研/竞赛队伍
- 兴趣运动队伍
- 自动匹配算法（按专业、时间、技能标签）

### 3.1.7 用户信用体系
信用规则：
- 完成任务 +1
- 被投诉 -2
- 迟到/爽约 -1

信用等级：优秀 / 良好 / 一般 / 需关注

影响：
- 可接单数量
- 是否需要额外验证

### 3.1.8 消息与通知系统
- 实时聊天
- 系统通知（任务状态变更）
- 订单提醒
- 举报处理结果通知

### 3.1.9 管理员后台
- 用户管理
- 任务审核
- 举报处理
- 数据统计（任务量、交易量、活跃度）

---

## 3.2 性能需求

### 3.2.1 响应时间
- 页面加载时间 ≤ 2 秒
- 消息延迟 ≤ 300 ms

### 3.2.2 并发能力
- 支持至少 10,000 名学生同时在线
- 高峰期任务发布量 ≥ 500 条/分钟

### 3.2.3 可用性
- 年可用性 ≥ 99%
- 支持自动重连机制

---

## 3.3 安全需求
- 使用 HTTPS
- 密码加密存储
- 敏感操作需二次验证
- 聊天内容加密传输
- 防止 SQL 注入、XSS、CSRF

---

## 3.4 软件质量属性
- **可维护性**：模块化设计，代码注释率 ≥ 20%
- **可扩展性**：支持未来接入 AI 辅助功能
- **可移植性**：支持 Web、iOS、Android
- **易用性**：界面简洁，操作步骤 ≤ 3 步

---

## 3.5 其他需求
- 支持深色模式
- 支持多语言（中文、英文）
- 支持校园地图定位
