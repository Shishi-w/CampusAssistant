## 一、项目角色分工表

| 角色 | 职责 | 姓名 |
|------|------|------|
| 需求负责人 | 负责需求获取、文档撰写、用例建模 | 陈一沛 |
| 架构负责人 | 负责技术选型、架构设计、ADR撰写 | 陈一沛 |
| 开发负责人 | 负责编码规范、代码审查、核心模块开发 | 冯子航 |
| 测试负责人 | 负责测试策略、测试用例设计、质量保障 | 王师师 |

#### 陈一沛：
需求负责人的工作核心的是需求全流程管控，日常对接用户，通过访谈、调研收集各类显性及隐性需求，避免理解偏差与遗漏。后续将零散需求整理为规范文档，明确优先级、验收标准与业务逻辑，并用用例图转化为可执行场景，减少团队沟通成本，同时跟进需求变更，把控流程以贴合业务目标。
架构负责人专注于项目技术架构设计与技术决策，结合项目并发量、安全性等需求，以及团队技术能力、成本预算，筛选适配的技术栈。在此基础上搭建整体架构，划分模块、明确交互逻辑与数据流转路径，设计核心技术方案，同时撰写ADR文档记录关键决策，把控技术方向，规避架构缺陷。
#### 冯子航：
开发板块的核心工作是带领团队将需求与架构设计落地为可运行代码。结合项目技术栈制定统一编码规范，明确代码格式、命名规则及注释要求，保障团队代码风格统一，提升可读性与可维护性。日常组织代码评审，排查代码规范、逻辑及性能问题，牵头开发核心模块与关键逻辑，攻克技术难点，指导团队成员开展工作、协调解决技术问题，合理分配任务，确保开发进度与质量，为测试、上线奠定基础。
#### 王师师：
测试负责人的核心职责是通过系统化测试保障软件产品质量，为上线提供支撑。结合需求文档与架构设计，制定全面测试策略，明确测试范围、类型、资源分配及进度计划，确保测试工作有序高效。基于需求与用例场景，设计覆盖正常、异常、边界等各类场景的测试用例，组织团队执行测试，跟踪缺陷从发现、提交到修复、回归的全流程，监控质量指标、排查质量风险并提出优化建议，为项目迭代提供质量反馈。

#### 组合方案（高级用法）

根据2026年行业实践，高效团队常采用分层审查策略：

```
Level 1 (编码时): Copilot 实时语法/规范检查
Level 2 (PR草稿): Copilot Agent 自动修复简单问题  
Level 3 (深度审查): CodeRabbit 分析架构和安全
Level 4 (人工把关): 工程师专注业务逻辑
```

**成本**：Cursor Pro (\$20) + CodeRabbit (\$24) + Copilot (\$10) ≈ **\$54/月/人**，适合专业开发者 。

## 最终选择方案组合

**AI编程助手**：Claude Code + 国产API  
**AI对话工具**：ChatGPT免费版
**AI代码审查工具**：CodeRabbit  
**AI测试工具**：webapp-test skill

### 选择理由

**Claude Code**：
- 强大的代码理解能力，支持多文件操作和执行测试
- 与终端深度集成，适合复杂的代码分析和修改任务
- 国产API提供了更稳定的访问方式和更好的响应速度，也较便宜

**ChatGPT**：
- 提供基础的文档生成和需求分析功能
- 零成本，适合日常轻量级任务
- 生态成熟，使用广泛

**CodeRabbit**：
- 市占率最高的专用AI代码审查工具（632,256+ PRs处理量）
- 深度上下文理解，基于代码图分析复杂依赖，80% Bug检测率领先
- 分层审查能力，区分潜在问题、重构建议和吹毛求疵

**Claude Code生成测试用例**：
- 利用Claude Code的代码理解能力，生成高质量的测试用例
- 与开发流程无缝集成，提高测试覆盖率
- 减少手动编写测试用例的工作量
### 不选择其他工具的原因

**GitHub Copilot**：虽然IDE支持广泛，但Premium Requests限制（Pro版300次/月）对重度用户不够用，且Agent Mode不够成熟。

**Cursor**：虽然多文件重构能力强，但信用系统导致费用不可预测，大型代码库性能下降明显。

**Gemini Code Assist**：生态较小，仅2,788个组织使用，社区和集成少于竞品，成熟度较低。

**SonarQube**：虽然稳定成熟，但基于规则而非AI，无法理解代码意图，缺乏跨服务分析能力。

**Testim AI/Diffblue Cover**：价格较高，学习曲线较陡，对于本项目规模来说过于复杂。
